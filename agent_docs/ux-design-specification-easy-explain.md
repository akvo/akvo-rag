# UX Design Specification: Simplified AI Responses

## Interaction Pattern: "Easy Explain" Follow-up

### Behavior
When a user asks for a simpler explanation (e.g., "explain in easy way"), the AI should:
1.  **Subject Persistence**: Retain the previous topic as the primary focus.
2.  **Tone & Style**:
    - Avoid jargon or define it simply if necessary.
    - Use short, punchy sentences.
    - Use relatable analogies (e.g., "Living income is like having enough money to not just survive, but to live with dignity").
3.  **Visual Structure**:
    - Use bullet points if listing multiple sub-points.
    - Keep the response to 2-3 short paragraphs maximum.
4.  **Citations**:
    - Maintain citations `[citation:x]` even in a simplified explanation.
    - Ensure citations point to the original source facts.

### Goal
The user should feel the AI "listens" to their stylistic preference without losing the factual grounding or getting distracted by implementation details.
